## 1. 项目背景与动机

分子性质预测是药物研发早期阶段的核心问题之一——在合成和实验测试之前，用计算方法预测分子的物理化学性质（如水溶性、脂溶性、毒性），可以大幅降低筛选成本。

本项目旨在构建一个**可复现、方法严谨的分子性质预测基准（benchmark）**，对比经典机器学习模型与神经网络模型在不同分子表示下的表现，并特别关注**评估方法的科学性**（scaffold-aware 验证），避免常见的"随机划分导致性能虚高"陷阱。

**核心价值主张**：不是简单调包跑分，而是展示对分子表示、验证方法论、模型选择的完整理解——这是这个项目区别于一般 ML 练手项目的地方。

---

## 2. 项目目标

### 2.1 主要目标
1. 搭建一个端到端 pipeline：原始分子数据 → 特征工程 → 模型训练 → 评估 → 可视化分析
2. 对比 **经典 ML（RF/XGBoost/SVM）** vs **神经网络（MLP/GNN）** 在分子性质回归任务上的表现
3. 验证并展示 **随机划分 vs scaffold 划分** 对模型评估结果的影响差异
4. 产出一个可扩展、文档清晰的开源项目，可放入作品集/简历

### 2.2 非目标（Out of Scope）
- 不追求在任何数据集上刷到 SOTA 精度
- 不实现复杂的预训练分子表示模型（如 ChemBERTa、MolCLR）——留作后续扩展
- 不涉及 3D 构象或量子化学计算性质（如 QM9 中的 HOMO/LUMO）
- 不做生产级部署（如 API 服务化）——本项目聚焦研究/分析

## 4. 数据集范围

### 4.1 第一阶段（MVP）
| 数据集 | 任务类型 | 分子数 | 性质 |
|---|---|---|---|
| ESOL (Delaney) | 回归 | 1128 | 水溶性 (log mol/L) |
| FreeSolv | 回归 | 642 | 水合自由能 (kcal/mol) |

选择理由：数据量小、CPU 可快速迭代、任务定义清晰、是 MoleculeNet 标准基准，方便与文献结果对比。

### 4.2 第二阶段（扩展，可选）
| 数据集 | 任务类型 | 分子数 | 性质 |
|---|---|---|---|
| Lipophilicity | 回归 | 4200 | 脂溶性 (logD) |
| BBBP | 分类 | ~2000 | 血脑屏障通透性 |
| BACE | 分类 | ~1500 | 酶抑制活性 |

扩展到分类任务可以补充分类指标（AUC、F1）的分析维度。

### 4.3 数据来源
- 原始 CSV：MoleculeNet 官方 (https://moleculenet.org/datasets-1)
- 备用加载方式：`deepchem.molnet` 或 `torch_geometric.datasets.MoleculeNet`

## 5. 功能需求

### 5.1 数据处理模块
- [ ] 下载/加载原始 CSV，统一格式为 `{smiles, target}`
- [ ] SMILES 有效性校验（用 RDKit 解析失败的分子需记录并剔除）
- [ ] 基础数据探索（分子量分布、原子数分布、目标值分布）

### 5.2 分子表示模块
- [ ] **RDKit 描述符**：分子量、LogP、TPSA、HBD/HBA、可旋转键数、环数等（约 20-30 个描述符）
- [ ] **Morgan 指纹（ECFP）**：可配置 radius（默认 2）和 bit 数（默认 2048）
- [ ] （可选）**图表示**：原子特征 + 邻接矩阵，供 GNN 使用
- [ ] 特征标准化（StandardScaler，仅对描述符，指纹不需要）

### 5.3 数据划分模块（核心亮点）
- [ ] **随机划分**：标准 train/val/test（如 80/10/10）
- [ ] **Scaffold 划分**：基于 Bemis-Murcko scaffold 分组，保证同一骨架不跨集合
- [ ] 划分结果可视化：训练/测试集的骨架重叠度、化学空间分布对比

### 5.4 模型模块
| 类别 | 模型 | 输入特征 |
|---|---|---|
| 经典 ML | Random Forest | 描述符 / 指纹 |
| 经典 ML | XGBoost | 描述符 / 指纹 |
| 经典 ML | Ridge / Lasso（线性基线） | 描述符 |
| 神经网络 | MLP | 描述符 / 指纹（拼接） |
| 神经网络（可选） | 简单 GCN | 图表示 |

- [ ] 统一的模型接口（fit/predict），方便批量跑 benchmark
- [ ] 超参数搜索（简单 grid search 或 Optuna，非必须但加分）

### 5.5 评估模块
- [ ] 回归指标：RMSE、MAE、R²
- [ ] 对比表格：数据集 × 特征类型 × 模型 × 划分方式 → 4维结果矩阵
- [ ] 统计显著性说明（如多次随机种子跑 5 次取均值±方差）

### 5.6 分析与可解释性模块
- [ ] 特征重要性（RF/XGBoost 内置 + SHAP 值）
- [ ] 预测值 vs 真实值散点图
- [ ] 误差分析：哪些分子/骨架预测误差大
- [ ] 化学空间可视化（UMAP/t-SNE 降维，按训练/测试集着色）
- [ ] 候选优先级排序 demo：结合预测值 + 不确定性（RF 树间方差）给出"建议优先测试"排序

### 5.7 交付物
- [ ] 结构清晰的代码仓库（见第 7 节）
- [ ] README：项目说明、结果摘要、如何复现
- [ ] Jupyter Notebook：完整分析流程 + 可视化
- [ ] 结果表格/图表（导出为图片或 HTML report）

## 6. 成功标准

| 维度 | 标准 |
|---|---|
| 功能完整性 | ESOL + FreeSolv 两个数据集，至少3种经典模型 + 1种神经网络模型跑通 |
| 方法严谨性 | 清晰展示 scaffold split 相比随机 split 的性能差异（预期 scaffold split 下 RMSE 更高，更接近真实泛化能力） |
| 结果可信度 | 与文献报告的 ESOL/FreeSolv baseline 结果量级一致（如 ESOL RF 通常 RMSE ~0.6-0.8 log units） |
| 代码质量 | 模块化、有注释、可复现（固定随机种子、requirements.txt） |
| 可展示性 | README + notebook 能让第三方在 10 分钟内理解项目做了什么、结论是什么 |

## 10. 风险与待解决问题

| 风险 | 应对方案 |
|---|---|
| 数据集小，神经网络容易过拟合 | 加正则化、dropout；强调这是预期现象，本身也是分析点 |
| Scaffold split 后某些骨架样本极少，划分不均衡 | 检查划分后训练/测试集大小分布，必要时用 scaffold 分组的分层策略 |
| RDKit 描述符/指纹选择过多导致维度灾难 | 先用小而精的描述符集合，指纹维度可调（512/1024/2048 对比） |
| 时间有限难以覆盖 GNN 部分 | GNN 作为 stretch goal，MVP 不依赖它 |
